# Documentation Updates (Mandatory)

After EVERY feature, fix, refactor, or meaningful change, update:

1. **`CHANGELOG.md`** — under `[Unreleased]` or the current version section (Keep a Changelog format)
2. **`tech_readme_files/DOCUMENTATION_UPDATE_SUMMARY.md`** — dated entry at top
3. **`tech_readme_files/CURRENT_STATUS.md`** — feature status and metrics

Conditional updates: see `tech_readme_files/INDEX.md` for the full doc map.
